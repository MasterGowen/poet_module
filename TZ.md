# Техническое задание: Модуль автономной генерации поэзии «Завод Навоза 2.0»

## 1. Общее описание
Модуль представляет собой автономное приложение на языке Python, предназначенное для автоматической генерации экзистенциальных стихов в специфическом стиле (смешение классической поэзии и абсурдного юмора с центральной темой «навоза/грязи»). 

Модуль должен работать независимо от основной инфраструктуры OpenClaw, используя прямое взаимодействие с API большой языковой модели (LLM).

## 2. Функциональные требования

### 2.1. Генерация контента (`generator.py`)
- **Взаимодействие с API:** Модуль должен отправлять HTTP-запросы к API LLM.
- **Управление промптами:** Система должна использовать системный промпт для обеспечения определенного стиля:
    - Смешение двух разных поэтов (один классик, один абсурдист/современник).
    - Обязательное присутствие темы «навоза», «грязи», «бетона» или «быта» как противопоставления высокому.
    - Создание структуры: Мотивы $\rightarrow$ Антонимы $\rightarrow$ Текст стихотворения.
- **Автономность:** Возможность работы без запущенного сервера OpenClaw.

### 2.2. Хранение данных (`recorder.py`)
- **Формат:** Хранение всех сгенерированных записей в формате JSON.
- **Путь:** Файл должен находиться строго внутри папки модуля: `data/records.json`.
- **Структура записи:**
    - `version`: Инкрементальный номер записи.
    - `date`: ISO-штамп времени генерации.
    - `poets`: Список задействованных стилей/авторов.
    - `motifs`: Список ключевых мотивов.
    - `antonyms`: Список противопоставлений.
    - `title`: Заголовок произведения.
    - `poem`: Текст стихотворения.
- **Безопасность:** Запрет на запись или изменение любых файлов вне директории `poet_module`.

### 2.3. Уведомления (`notifier.py`)
- **Текущая реализация:** Вывод результата генерации в стандартный поток вывода (stdout) с пометкой `[TELEGRAM STUB]`.
- **Архитектура:** Модуль должен быть спроектирован так, чтобы в будущем легко заменить заглушку на реальный API Telegram.

### 2.4. Управление и координация (`main.py`)
- **Поток выполнения:** `Запуск` $\rightarrow$ `Загрузка конфига` $\rightarrow$ `Генерация стиха` $\rightarrow$ `Запись в JSON` $\rightarrow$ `Уведомление`.

## 3. Технические требования

### 3.1. Стек технологий
- **Язык:** Python 3.10+.
- **Библиотеки:** `requests` (для API), `json`, `datetime`.
- **Контроль версий:** Git.

### 3.2. Конфигурация (`config.json`)
Файл должен содержать:
- `api_key`: Ключ доступа к LLM.
- `api_endpoint`: URL эндпоинта.
- `model_name`: Идентификатор используемой модели.
- `paths`: Пути к файлам данных.

## 4. Структура проекта
```text
poet_module/
├── .git/                # Git репозиторий
├── config.json          # Конфигурационный файл
├── data/
│   └── records.json     # Архив стихов
├── src/
│   ├── generator.py     # Логика работы с ИИ
│   ├── recorder.py      # Логика сохранения
│   ├── notifier.py      # Логика уведомлений
│   └── main.py         # Точка входа
└── README.md            # Документация
```

## 5. Критерии приемки
1. Скрипт запускается командой `python3 src/main.py`.
2. В `data/records.json` появляется новая запись с корректной структурой.
3. В консоли отображается сгенерированный стих.
4. В корневом воркспейсе (`/home/gowen/.openclaw/workspace/`) не произошло никаких изменений.

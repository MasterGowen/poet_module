# Завод Навоза 2.0 — Автономный генератор экзистенциальной поэзии

## Описание

Модуль генерирует стихи в стиле «классика × абсурд» через LLM API.
Центральный образ — **говно** как радикальная метафора (валюта, божество, язык, архитектурный фундамент). Форма свободная, но подчинённая жёсткой внутренней логике: верлибр, внутренняя рифма, аллитерация, энжамбеман, синтаксический параллелизм.

## Запуск

```bash
python3 -m src.main
```

## Требования

- Python 3.10+
- `.env` с обязательными полями: `API_KEY`, `API_ENDPOINT`, `MODEL_NAME`
- Зависимости: `httpx`, `pydantic-settings`, `filelock` (см. `requirements.txt`)

## Структура

- `src/generator.py`: **API LLM** — взаимодействие с моделью (`httpx.Client`), промпт-инженерия, парсинг JSON-ответа из `<json>` тегов. Реализованы ретраи с exponential backoff (3 попытки, таймаут 120с) при ошибках `429`/`5xx`. Корректировка урезанных ключей (`po` → `poem`).
- `src/recorder.py`: **Атомарная запись** — `FileLock` для защиты от concurrent access, валидация путей через `os.path.commonpath()` (изоляция от `poet_module/`), атомарная запись через `temp → os.replace`. przy сбоях — бэкап испорченного файла в `.corrupt.{timestamp}.json`. Монотонное версионирование.
- `src/notifier.py`: **Уведомления** — заглушка (`stdout`), архитектура на абстрактном классе `BaseNotifier` для будущей замены на Telegram API.
- `src/main.py`: **Оркестрация** — DI компонентов, логирование в `data/app.log` и `data/error.log`, обработка критических ошибок.

## Конфигурация

Файл `.env`:

```ini
API_KEY=<ключ доступа>
API_ENDPOINT=<URL эндпоинта>
MODEL_NAME=<название модели>
```

## данные

- `data/records.json` — архив стихов (атомарная запись)
- `data/app.log` — логи приложения
- `data/error.log` — логи ошибок

## Безопасность

- `.env` в `.gitignore`
- Валидация путей: любая попытка записи вне `poet_module/` вызывает `PermissionError`
- Атомарная запись гарантирует целостность `records.json` даже при `kill -9`

## Критерии приёмки

1. **Безопасность**: API-ключ не попадает в Git.
2. **Целостность**: Принудительное завершение (`kill -9`) не портит `records.json`.
3. **Изоляция**: Запись вне папки модуля bloqueada.
4. **Монотонность**: Каждая запись имеет строго следующий номер версии.
5. **Валидация**: Скрипт не запустится без заполненного `.env`.
